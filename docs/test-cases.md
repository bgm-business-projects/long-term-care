## 測試案例

### TC-01：儀表板總覽 (B01-01)

**對應需求：** 登入後首頁顯示今日訂單總數、待派訂單數、執行中車輛數

**測試步驟：**
1. 以 admin 帳號登入
2. 呼叫 `GET /api/dispatch/dashboard?date=2026-03-17`

**預期結果：**
- 今日訂單總數：9 筆（不含昨日）
- 待派訂單數：5 筆（Trip 3, 4, 7, 8, 9）
- 進行中車輛數：1 台（Trip 2 DEF-5678）
- 已完成訂單：1 筆（Trip 6）

---

### TC-02：甘特圖排班台 (B04-01)

**對應需求：** 橫軸為時間，縱軸為車輛，以色塊顯示已被佔用的時段

**測試步驟：**
1. 呼叫 `GET /api/dispatch/schedule/gantt?date=2026-03-17`

**預期結果：**
- ABC-1234：08:00-08:30 已佔用（陳爺爺）
- DEF-5678：07:30-08:05 已佔用（林奶奶）
- GHI-9012：08:30-08:55 已佔用（王奶奶）
- JKL-3456：13:30-14:10 已佔用（劉爺爺）
- MNO-7890：全天空閒
- 右側待排池：顯示 5 筆 pending 訂單

---

### TC-03：拖曳指派 — 正常指派 (B04-02)

**對應需求：** 將待派訂單拖拉至空閒車輛的時段中

**測試步驟：**
1. 選取 Trip 3（黃伯伯 09:00 去榮總）
2. 呼叫 `PATCH /api/dispatch/trips/{trip3_id}/assign`
   ```json
   {
     "vehicleId": "{ABC-1234_id}",
     "driverUserId": "{陳大偉_id}",
     "scheduledAt": "2026-03-17T09:00:00",
     "scheduledEndAt": "2026-03-17T10:00:00"
   }
   ```

**預期結果：**
- 狀態碼 200
- Trip 3 狀態變為 `assigned`
- 甘特圖 ABC-1234 新增 09:00-10:00 色塊

---

### TC-04：資源衝突檢核 — 時間重疊 (B04-03)

**對應需求：** 當訂單被拖入時，系統檢查時間重疊衝突

**測試步驟：**
1. 選取 Trip 9（陳爺爺 16:00 返家）
2. 嘗試指派到 ABC-1234，但故意設定時間與 Trip 1 重疊
   ```json
   {
     "vehicleId": "{ABC-1234_id}",
     "driverUserId": "{陳大偉_id}",
     "scheduledAt": "2026-03-17T08:00:00",
     "scheduledEndAt": "2026-03-17T08:30:00"
   }
   ```

**預期結果：**
- 狀態碼 409（Conflict）
- 回傳衝突詳情，包含 Trip 1 資訊
- 前端顯示衝突警示

---

### TC-05：資源衝突檢核 — 輪椅車空間不足 (B04-03)

**對應需求：** 檢查車輛輪椅空間是否足夠

**測試步驟：**
1. 選取 Trip 4（張阿嬤 10:00，需輪椅）
2. 嘗試指派到 GHI-9012（轎車，無輪椅升降）
   ```json
   {
     "vehicleId": "{GHI-9012_id}",
     "driverUserId": "{吳建國_id}",
     "scheduledAt": "2026-03-17T10:00:00",
     "scheduledEndAt": "2026-03-17T10:45:00"
   }
   ```

**預期結果：**
- 應檢查輪椅需求與車輛能力是否匹配
- 如系統實作了輪椅檢查：應回傳警告或拒絕
- 如尚未實作：此為待補功能的驗證案例

### TC-08：司機狀態回報流程 (F02-03)

   ```
   POST /api/driver/trips/{trip1_id}/status
   { "logStatus": "arrived_origin", "lat": 25.0425, "lng": 121.5135 }

   ```
   ```
   POST /api/driver/trips/{trip1_id}/status
   ```

**預期結果：**
- 每次回報記錄時間戳和 GPS 座標


### TC-16：公告管理（發布/取消發布）

**對應需求：** 管理員發布通知給司機

**測試步驟：**
1. 取得草稿公告（端午排班）ID
2. 呼叫 `POST /api/dispatch/announcements/{id}/publish`
3. 驗證司機端可看到
4. 呼叫 `POST /api/dispatch/announcements/{id}/unpublish`
5. 驗證司機端不可看到

**預期結果：**
- publish 後 `isPublished: true`，司機端可見
- unpublish 後 `isPublished: false`，司機端不可見
